# 后端 AGENT

本文件是后端 AI 的最小约束入口，只放关键规则。详细规范按指引到 `docs/backend/文档索引.md` 查阅。

## 角色

你负责 `backend/` 内的 Java + Spring Boot + Maven 后端工程，重点保证分层清晰、接口稳定、数据库设计可演进。

## 必查文档

- 后端索引：`docs/backend/文档索引.md`
- 后端详细规范：`docs/backend/后端工程规范.md`
- 项目需求：`require/全国旅游地图项目.md`
- 数据库草案：`require/数据库设计.md`
- 地图说明：`docs/地图API接入说明.md`

## 硬性约束

- URL 使用 `/api` 前缀。
- Controller 不写复杂业务逻辑，业务逻辑放 Service，数据库访问放 Mapper。
- 涉及筛选、排序、分页、聚合、统计等和真实数据相关的计算，默认应在后端提供参数和处理逻辑，避免把数据处理压力转嫁给前端。
- 实体字段变更必须同步数据库设计或相关文档。
- 更新相关业务代码时，必须同步更新对应说明文档，避免接口、数据库、配置说明和实际实现不一致。
- 不把数据库密码、地图服务端 Key 等敏感配置写死到源码。
- 编写类、方法和重要业务步骤时添加通俗易懂中文注释。

## 工作流程

- 动手前先判断任务属于哪个开发阶段，必要时查看 `docs/开发规划.md`。
- 需求、数据库设计、当前阶段冲突时，先提醒并说明影响。
- 如果改动涉及数据库、接口、配置、部署方式或开发流程，必须检查并同步更新相关文档。
- 每次完成后说明改了哪些文件、如何运行和如何验证。
- 如需详细包结构、接口、数据库、地图 API 规则，查看 `docs/backend/后端工程规范.md`，不要把详细规则复制回本文件。

## 质量门禁

后端功能改动后至少执行：

```bash
mvn test
```

如果影响前端联调，再补充前端 `npm run build`
