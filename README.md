# 🗺️ 行迹旅行地图 (Trail Map)

行迹旅行地图是一个主打“地图可视化”的全国旅游工具应用。本项目旨在帮助用户直观地查看城市中最值得游览的景点，提供真实的行程规划工具，并记录个人的旅行足迹。

## ✨ 核心功能模块

### 📍 沉浸式地图探索与景点发现
* **多级地图展示**：在全国地图基础上支持层级缩放，低层级聚合展示热门区域（如西安的钟楼商圈、曲江区域），高层级展示具体景点点位。
* **场景化智能筛选**：提供“适合首次来”、“雨天可去”、“地铁方便”等多维度标签筛选，拒绝单一的历史/自然分类。

### 🛣️ 双模式智能路线规划
依托真实的地图路线规划工具，不仅提供简单的游玩顺序，更能精细到交通方式及耗时。
* **自由路线模式**：随性游玩首选。根据所选景点和起点，自动输出推荐游玩顺序、各段交通耗时及景点建议停留时间。
* **完整行程模式**：支持设置起止时间、是否安排午餐/夜游及游玩强度（轻松/紧凑），系统自动为您拆分每日（Day 1, Day 2）精细化行程。

### 🎒 待规划景点池
* 从地图上将心仪景点“加入行程”，右侧生成专属景点池。
* 支持拖拽调整顺序、一键智能排序、删除及将某景点设为终点。

### 👣 个人旅行足迹系统
* 支持用户账号登录，记录个人的旅游足迹地图。
* 已打卡的景点可在地图上置灰或隐藏，避免视觉干扰，并支持标记“不感兴趣”以优化推荐。

## 🛠️ 技术架构

本项目采用前后端分离架构，主要技术栈如下：

### 前端 (Frontend)
* **核心框架**：React + TypeScript + Vite
* **状态管理**：Zustand + TanStack Query
* **UI 组件库**：Ant Design + CSS Modules
* **地图双引擎**：百度地图 JS API（负责路线规划、POI搜索） + AntV L7（负责足迹、聚合等地理数据可视化）

### 后端 (Backend)
* **核心框架**：Java + Spring Boot
* **持久层**：MyBatis-Plus + MySQL
* **API 文档**：Knife4j / Swagger
* **数据库迁移**：Flyway (通过 `V*` 迁移脚本演进)
