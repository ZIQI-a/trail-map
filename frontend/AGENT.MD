# 前端 AGENT

本文件是前端 AI 的最小约束入口，只放关键规则。详细规范按指引到 `docs/frontend/文档索引.md` 查阅。

## 角色

负责 `frontend/` 内的 React + TypeScript + Vite 前端工程。开发时要兼顾 Web 端体验和后续 H5、多端扩展。

## 硬性约束

- 不把百度地图 Key、后端地址等敏感配置写死到源码。
- 当前项目已进入百度地图接入阶段；涉及地图能力时，优先复用现有地图适配层和相关组件，不要退回到新的纯 Mock 方案。
- AntV L7 仅用于足迹、聚合、热力等地理数据可视化；路线规划、POI 搜索和选点仍复用百度地图，详细规则见 `docs/frontend/AntV-L7使用说明.md`。
- 页面要考虑 H5 兼容，小屏不能横向溢出。
- 路由级页面统一放到 `src/pages`；页面组件放到 `src/components`；Mock 数据放到 `src/mocks`；工具函数放到 `src/utils`。
- 页面组件只做布局编排和跨组件状态，不把多个小模块长期堆在一个主页面文件里。
- 管理端模块单独收拢到独立目录，避免和地图工作台等用户端模块混放。
- 每次改动以当前工作区代码现状为准；如果发现此前改过的参数或结构又发生变化，默认视为用户调整，不追溯、不回滚，除非它直接阻塞本次任务。
- 已引入 Ant Design，通用表单、弹窗、空状态等基础 UI 可优先使用组件库；若组件库默认样式或切换动效不符合原型和产品体验，允许使用原生元素 + CSS Modules 自定义实现。
- 接口请求统一放到 `src/api`，通用请求封装放到 `src/lib`，组件内不直接散写复杂请求逻辑。
- 涉及筛选、排序、分页、聚合、统计等和真实数据相关的计算时，默认优先由后端提供参数并处理，前端不要长期承担数据计算压力。
- 业务组件优先使用 CSS Modules，并保持组件 Props 类型明确。
- 编写函数、组件和重要步骤时添加简洁中文注释；新增 `function` 方法必须有函数级注释，说明职责和关键输入输出。
- 组件文件避免长期堆放 interface、常量映射、颜色映射、数据转换等纯逻辑；可复用或成组逻辑应按职责拆到同级 `types.ts`、`constants.ts`、`utils.ts` 或领域数据文件中。
- 前端页面不展示“后续开发”、“即将开放”、“开发说明”一类面向开发过程的解释性文案。
- 远程接口失败时不得用大块 `div`、`Alert` 或整页错误状态替换原有页面；应尽量保留页面结构、地图等可独立工作的能力和已有数据。一般错误使用 `message`、`notification` 或不影响布局的紧凑悬浮提示，字段错误直接绑定到对应表单项并标红；只有页面确实无法安全渲染时才允许使用阻断式错误页。
- 更新业务代码时，同步更新相关文档，避免规范、阶段说明和实际实现不一致。
- 默认不主动启动本地开发服务；页面预览由用户本地运行，AI 只执行质量验证。

## 工作流程

- 动手前先判断任务属于哪个开发阶段，必要时查看 `docs/开发规划.md`。
- 需求、原型、规范冲突时，先提醒并说明影响。
- 每次完成后说明改了哪些文件、如何运行和如何验证。
- 如需详细目录、组件、样式、接口规则，查看 `docs/frontend/前端工程规范.md`，不要把详细规则复制回本文件。
- 除非用户明确要求，不执行 `npm run dev` 或尝试占用本地端口。

## 质量门禁

每次功能改动后至少执行：

```bash
npm run lint
npm run build
```

如果这两步不过，不应视为改动完成。
