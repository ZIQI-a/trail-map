# 一、数据库表总览
## MVP 必做表
```plain
city                 城市表
spot                 景点表
spot_tag             景点标签表
spot_tag_relation    景点-标签关联表
spot_image           景点图片表
travel_area          城市热门游玩区域表
travel_area_spot     区域-景点关联表
route_record         路线规划记录表
route_segment        路线分段表
```

## 二期用户系统表
```plain
app_user             用户表
user_favorite_spot   用户收藏景点表
user_checkin_spot    用户打卡景点表
user_trip            用户保存行程表
user_trip_spot       用户行程景点表
```



# 二、MVP 核心表设计
---

## 1. city 城市表
用于存储项目支持的旅游城市。

| 字段名 | 类型 | 说明 |
| --- | --- | --- |
| id | BIGINT | 主键 |
| city_name | VARCHAR(50) | 城市名称 |
| province_name | VARCHAR(50) | 所属省份 |
| city_code | VARCHAR(50) | 城市编码，可用于地图 API |
| center_lng | DECIMAL(10,6) | 城市中心经度 |
| center_lat | DECIMAL(10,6) | 城市中心纬度 |
| map_zoom | INT | 默认地图缩放级别 |
| cover_url | VARCHAR(255) | 城市封面图 |
| description | VARCHAR(500) | 城市简介 |
| recommend_days | DECIMAL(3,1) | 推荐游玩天数 |
| hot_score | INT | 城市热度 |
| sort_order | INT | 排序 |
| status | TINYINT | 状态：1 启用，0 禁用 |
| created_at | DATETIME | 创建时间 |
| updated_at | DATETIME | 更新时间 |


示例：

```plain
成都 / 四川省 / 104.066541 / 30.572269 / 推荐 3 天
```

---

## 2. spot 景点表
这是最核心的表。

| 字段名 | 类型 | 说明 |
| --- | --- | --- |
| id | BIGINT | 主键 |
| city_id | BIGINT | 所属城市 ID |
| spot_name | VARCHAR(100) | 景点名称 |
| spot_type | VARCHAR(50) | 景点类型 |
| lng | DECIMAL(10,6) | 经度 |
| lat | DECIMAL(10,6) | 纬度 |
| address | VARCHAR(255) | 地址 |
| amap_poi_id | VARCHAR(100) | 高德 POI ID，可选 |
| boundary_geojson | LONGTEXT | 景点轮廓 GeoJSON，可选，适合古街、商圈、景区等区域型景点 |
| cover_url | VARCHAR(255) | 景点封面图 |
| summary | VARCHAR(500) | 简短介绍 |
| description | TEXT | 详细介绍 |
| recommend_reason | TEXT | 推荐理由 |
| travel_guide | TEXT | 游玩攻略 |
| opening_hours | VARCHAR(255) | 开放时间 |
| ticket_info | VARCHAR(255) | 门票信息 |
| suggested_duration | INT | 建议游玩时长，单位分钟 |
| best_time | VARCHAR(100) | 推荐游玩时间 |
| recommend_score | DECIMAL(2,1) | 推荐指数，例如 4.8 |
| hot_score | INT | 热度 |
| suitable_crowd | VARCHAR(255) | 适合人群 |
| is_free | TINYINT | 是否免费 |
| is_indoor | TINYINT | 是否室内 |
| is_night | TINYINT | 是否适合夜游 |
| is_rainy_day | TINYINT | 是否雨天可去 |
| subway_friendly | TINYINT | 是否地铁方便 |
| first_visit | TINYINT | 是否适合首次来 |
| sort_order | INT | 排序 |
| status | TINYINT | 状态：1 启用，0 禁用 |
| created_at | DATETIME | 创建时间 |
| updated_at | DATETIME | 更新时间 |


---

spot_type 建议枚举

```plain
history        历史文化
nature         自然风光
landmark       城市地标
museum         博物馆展馆
food           美食街区
night          夜游景点
family         亲子游玩
business       商圈街区
```

数据库里可以先用字符串，后端用枚举管理。

---

### 3. spot_tag 景点标签表
用于维护景点标签。

| 字段名 | 类型 | 说明 |
| --- | --- | --- |
| id | BIGINT | 主键 |
| tag_name | VARCHAR(50) | 标签名称 |
| tag_code | VARCHAR(50) | 标签编码 |
| tag_type | VARCHAR(50) | 标签类型 |
| sort_order | INT | 排序 |
| status | TINYINT | 状态 |
| created_at | DATETIME | 创建时间 |
| updated_at | DATETIME | 更新时间 |


示例标签：

```plain
适合拍照
适合夜游
免费景点
地铁方便
雨天可去
亲子友好
适合情侣
半日游
首次必去
```

---

### 4. spot_tag_relation 景点标签关联表
景点和标签是多对多关系。

| 字段名 | 类型 | 说明 |
| --- | --- | --- |
| id | BIGINT | 主键 |
| spot_id | BIGINT | 景点 ID |
| tag_id | BIGINT | 标签 ID |
| created_at | DATETIME | 创建时间 |


一个景点可以有多个标签，例如：

```plain
宽窄巷子：
适合首次来 / 适合拍照 / 美食街区 / 地铁方便
```

---

### 5. spot_image 景点图片表
一个景点可能有多张图片。

| 字段名 | 类型 | 说明 |
| --- | --- | --- |
| id | BIGINT | 主键 |
| spot_id | BIGINT | 景点 ID |
| image_url | VARCHAR(255) | 图片地址 |
| image_type | VARCHAR(50) | 图片类型 |
| sort_order | INT | 排序 |
| status | TINYINT | 状态 |
| created_at | DATETIME | 创建时间 |


image_type 可以是：

```plain
cover       封面图
gallery     详情图
map_card    地图卡片图
```

MVP 也可以先只用 spot 表里的 `cover_url`，后期再用这张表扩展多图。

---

## 6. travel_area 城市热门游玩区域表
用于表达“区域玩法”，比如成都的“宽窄巷子-人民公园区域”。

| 字段名 | 类型 | 说明 |
| --- | --- | --- |
| id | BIGINT | 主键 |
| city_id | BIGINT | 城市 ID |
| area_name | VARCHAR(100) | 区域名称 |
| center_lng | DECIMAL(10,6) | 区域中心经度 |
| center_lat | DECIMAL(10,6) | 区域中心纬度 |
| cover_url | VARCHAR(255) | 区域封面图 |
| summary | VARCHAR(500) | 区域简介 |
| description | TEXT | 区域详细介绍 |
| suggested_duration | INT | 建议游玩时长，单位分钟 |
| best_time | VARCHAR(100) | 适合游玩时间 |
| recommend_reason | TEXT | 推荐理由 |
| sort_order | INT | 排序 |
| status | TINYINT | 状态 |
| created_at | DATETIME | 创建时间 |
| updated_at | DATETIME | 更新时间 |


示例：

```plain
成都：
宽窄巷子-人民公园区域
武侯祠-锦里区域
春熙路-太古里区域
熊猫基地区域
```

---

### 7. travel_area_spot 区域景点关联表
一个区域下可以包含多个景点。

| 字段名 | 类型 | 说明 |
| --- | --- | --- |
| id | BIGINT | 主键 |
| area_id | BIGINT | 区域 ID |
| spot_id | BIGINT | 景点 ID |
| sort_order | INT | 区域内景点排序 |
| created_at | DATETIME | 创建时间 |


---

## 8. route_record 路线规划记录表
用于保存一次路线规划的总信息。

MVP 阶段即使没有用户系统，也可以保存匿名路线记录，方便后续展示、调试、缓存。

| 字段名 | 类型 | 说明 |
| --- | --- | --- |
| id | BIGINT | 主键 |
| city_id | BIGINT | 城市 ID |
| user_id | BIGINT | 用户 ID，MVP 可为空 |
| start_name | VARCHAR(100) | 起点名称 |
| start_lng | DECIMAL(10,6) | 起点经度 |
| start_lat | DECIMAL(10,6) | 起点纬度 |
| end_name | VARCHAR(100) | 终点名称 |
| end_lng | DECIMAL(10,6) | 终点经度 |
| end_lat | DECIMAL(10,6) | 终点纬度 |
| transport_type | VARCHAR(50) | 交通方式 |
| plan_mode | VARCHAR(50) | 规划模式 |
| spot_ids | VARCHAR(500) | 途经景点 ID 列表 |
| total_distance | INT | 总距离，单位米 |
| total_duration | INT | 总耗时，单位秒 |
| route_summary | VARCHAR(500) | 路线摘要 |
| raw_request | JSON | 原始请求参数 |
| raw_response | JSON | 地图 API 原始响应 |
| status | TINYINT | 状态 |
| created_at | DATETIME | 创建时间 |
| updated_at | DATETIME | 更新时间 |


---

### transport_type 建议枚举
```plain
walking     步行
driving     驾车
transit     公共交通
bicycling   骑行
```

### plan_mode 建议枚举
```plain
free        自由路线模式
schedule    完整行程模式
```

---

### 9. route_segment 路线分段表
用于保存路线中的每一段。

例如：

```plain
酒店 → 宽窄巷子
宽窄巷子 → 人民公园
人民公园 → 武侯祠
```

每一段就是一条 route_segment。

| 字段名 | 类型 | 说明 |
| --- | --- | --- |
| id | BIGINT | 主键 |
| route_record_id | BIGINT | 路线记录 ID |
| segment_index | INT | 第几段 |
| from_name | VARCHAR(100) | 起点名称 |
| from_lng | DECIMAL(10,6) | 起点经度 |
| from_lat | DECIMAL(10,6) | 起点纬度 |
| to_name | VARCHAR(100) | 终点名称 |
| to_lng | DECIMAL(10,6) | 终点经度 |
| to_lat | DECIMAL(10,6) | 终点纬度 |
| transport_type | VARCHAR(50) | 交通方式 |
| distance | INT | 距离，单位米 |
| duration | INT | 耗时，单位秒 |
| instruction | TEXT | 路线说明 |
| polyline | LONGTEXT | 路线坐标串 |
| steps_json | JSON | 详细步骤 JSON |
| created_at | DATETIME | 创建时间 |


这个表很重要，因为你的路线展示需要分段。

### 10. app_user 用户表

实际开发中不直接使用 `user` 作为表名，避免和数据库系统用户、保留语义或运维工具产生歧义。

| 字段名 | 类型 | 说明 |
| --- | --- | --- |
| id | BIGINT | 主键 |
| username | VARCHAR(50) | 用户名 |
| nickname | VARCHAR(50) | 昵称 |
| password_hash | VARCHAR(255) | 密码哈希，不保存明文 |
| avatar_url | VARCHAR(255) | 头像 |
| phone | VARCHAR(20) | 手机号 |
| email | VARCHAR(100) | 邮箱 |
| status | TINYINT | 状态 |
| last_login_at | DATETIME | 最近登录时间 |
| created_at | DATETIME | 创建时间 |
| updated_at | DATETIME | 更新时间 |


MVP 不做登录的话，这张表先不建也可以。

# 四、核心关系图
可以理解成：

```plain
city
  └── spot
        ├── spot_image
        └── spot_tag_relation
                └── spot_tag

city
  └── travel_area
        └── travel_area_spot
                └── spot

route_record
  └── route_segment

app_user
  ├── user_favorite_spot
  ├── user_checkin_spot
  └── user_trip
        └── user_trip_spot
```
