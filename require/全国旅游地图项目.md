# 需求分析
主打内容：提供一个供用户查看该城市或区域最值得去的景点的地图，作为旅游地点查看使用；

包括用户行程规划，行程规划为真实的行程规划，从起点出发选择交通方式展示具体的行程、时间等信息，具体到地铁站做几站（类似于高德地图等路线规划）

用户账号功能，登录可点击打卡去过的地方，打卡后该地点就去掉不干扰用户；

## 两条路径：
进入首页 → 选择城市 → 地图定位到城市 → 展示景点点位 → 点击景点 → 查看详情和攻略 → 收藏/加入行程

> 我想知道这个城市有哪些值得去的地方，它们都分布在哪里。
>



选择城市 → 选择出发地 → 选择游玩天数 → 选择想去的景点 → 选择交通方式 → 生成行程路线 → 查看每天的路线顺序、交通时间、游玩时间

> 我已经想去某个城市了，帮我安排怎么玩。
>



## 模块规划
### 首页：
展示全国地图，可缩放或点击，放大到城市后展示景点，

抽屉展示景点信息；

用户定位，可直接导航到景点；

上方城市选择、景点类型筛选等功能

#### 景点筛选
比如：

```plain
适合首次来
适合拍照
适合夜游
适合亲子
适合情侣
免费景点
室内景点
雨天可去
半日游
地铁方便
```

这比单纯的：

```plain
历史 / 自然 / 博物馆
```

#### 地图展示层级
缩放层级低

只显示：

```plain
城市热门区域
热门景点聚合点
```

例如：

```plain
西安：
钟楼商圈
大雁塔区域
曲江区域
临潼区域
```

缩放层级高

才显示具体景点：

```plain
钟楼
鼓楼
回民街
城墙
大雁塔
大唐不夜城
```

### 旅途规划：
选择去哪，几天，什么时候出发，交通方式后开始规划；

具体规划，不要假大空，不用借助 ai，直接使用规划工具例如高德地图来执行路线，不是那种说早上几点到、然后晚上睡一觉这样的，是路线规划；

所以两个路径规划方式：

1、自由路线模式

适合喜欢随性玩的用户。

用户只选择：

```plain
城市
起点
想去的景点
交通方式
```

系统输出：

```plain
推荐游玩顺序
每段交通方式
每段路线耗时
总路线长度
每个景点建议停留时间
```

但不强制安排几点到几点。

示例：

```plain
路线顺序：
酒店 → 陕西历史博物馆 → 大雁塔 → 大唐芙蓉园 → 大唐不夜城

交通：
酒店 → 陕西历史博物馆：地铁 2 号线换乘 3 号线，约 28 分钟
陕西历史博物馆 → 大雁塔：步行约 15 分钟
大雁塔 → 大唐芙蓉园：打车约 10 分钟
大唐芙蓉园 → 大唐不夜城：步行约 12 分钟
```



2、完整行程模式

用户额外选择：

```plain
每天开始时间
每天结束时间
是否安排午餐
是否安排夜游
游玩强度：轻松 / 标准 / 紧凑
```

系统输出：

```plain
Day 1
09:00 出发
09:30 到达第一个景点
建议停留 2 小时
11:30 前往下一个景点
...
```

这个模式可以做，但不要作为默认模式。



最后既然有路线规划，那么也就有去哪个景点；

### 景点池
用户在地图上看到景点后，可以点“加入行程”。

右侧形成一个“待规划景点池”。

比如：

```plain
已选择 5 个景点：
宽窄巷子
人民公园
武侯祠
锦里
太古里
```

用户可以：

```plain
删除景点
拖拽调整顺序
一键智能排序
选择某个景点作为终点
清空景点池
```

---

至此，路线规划分三个层级；

1、单点导航，选择景点（地点），可以直接导航到目的地；

2、多点导航：用户选择多个景点，系统根据位置顺序或用户手动顺序串联路线。

3、多行程模式：用户选择游玩 2 天或 3 天后，系统自动拆分：

```plain
Day 1：市中心景点
Day 2：远一点的景点
Day 3：轻松补充路线
```



路线偏好功能？

```xml
出行方式：
公共交通 / 驾车 / 步行 / 骑行 / 打车

路线偏好：
时间最短
步行最少
换乘最少
费用最低
景点顺路优先
轻松游玩
紧凑游玩
```



## 后期规划
以下功能第一版不急着做；

1、区域玩法功能

比如西安：

```plain
钟楼-回民街区域
大雁塔-大唐不夜城区域
兵马俑-华清宫区域
```

成都：

```plain
宽窄巷子-人民公园区域
武侯祠-锦里区域
春熙路-太古里区域
熊猫基地区域
```

用户点一个区域，不是看单个景点，而是看：

```plain
这个区域适合玩多久
区域内有哪些景点
推荐游玩路线
适合白天还是晚上
附近交通情况
```

2、用户登录系统：旅行足迹

功能：

```plain
去过的景点变灰
去过的景点可隐藏
城市完成度
省份点亮
我的足迹地图
收藏想去
已去 / 想去 / 不感兴趣
```

其中“不感兴趣”也很有用。

用户不只是去过某个地方，也可能是：

```plain
我不想去这个景点，以后别推荐了。
```
