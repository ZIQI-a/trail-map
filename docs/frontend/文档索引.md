# 前端文档索引

本文件是前端 Harness 工程入口。AI 在处理 `frontend/` 目录任务时，应先读取 `frontend/AGENT.MD`，再按任务需要查阅本索引中的详细文档。

- 项目总需求：`require/全国旅游地图项目.md`

- 分阶段计划：`docs/开发规划.md`

## 详细规范

- 前端工程规范：`docs/frontend/前端工程规范.md`

## 当前阶段关注点

- 当前已完成阶段 2、阶段 4、阶段 5、阶段 6 / 6.5 的主体工作，正在推进阶段 7：用户登录与旅行足迹。
- 地图工作台主数据源已切换到后端接口，百度地图也已接入，当前重点是登录态、收藏、打卡和行程保存。
- 前端请求层已支持自动携带登录 token；新增用户能力时优先复用 `frontend/src/lib/http.ts`。
- 地图相关开发应优先基于现有百度地图组件、路线绘制和 POI 联想能力继续迭代。
- 管理端当前已从“用户管理”扩展到“城市管理、景点管理”基础维护能力；新增后台模块时优先复用现有 admin 页面框架、查询 Hook 和统一表单弹窗模式。

## 查阅指引

- 做页面布局前：查看原型图和 `docs/frontend/前端工程规范.md`。
- 做阶段 2 任务前：查看 `docs/frontend/阶段2-前端静态地图工作台拆分计划.md`。
- 做接口调用前：查看后端接口文档；如果不存在，应先补接口契约文档。
- 做地图相关功能前：查看 `docs/地图API接入说明.md`。
- 修改技术栈或目录结构前：先确认 `docs/开发规划.md` 是否需要同步更新。
