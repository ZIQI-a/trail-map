## 当前阶段关注点

- 当前已完成阶段 2、阶段 4、阶段 5、阶段 6 / 6.5，阶段 7 的主体功能也已落地，当前重点转为功能优化与体验打磨。
- 地图工作台主数据源已切换到后端接口，百度地图也已接入，当前重点是登录态、收藏、打卡和行程保存。
- 前端请求层已支持自动携带登录 token；新增用户能力时优先复用 `frontend/src/lib/http.ts`。
- 已新增独立收藏页 `/favorites`，当前接入景点收藏列表；路线和城市收藏视觉入口保留，暂不返回真实数据。
- 行程分享链接使用 `/?shareToken=xxx`，由公开行程接口加载；私有的 `/?tripId=xxx` 仍只用于登录用户自己的行程回放。
- 已开始接入独立足迹页 `/checkins`，足迹地图可视化使用 AntV L7，具体约束见 `docs/frontend/AntV-L7使用说明.md`。
- 地图相关开发应优先基于现有百度地图组件、路线绘制和 POI 联想能力继续迭代。
- 地图工作台在后端数据接口不可用时仍保留基础底图、顶部导航和规划区，错误使用非阻断提示，不再展示占满主体区域的错误卡片。
- 管理端当前已从“用户管理”扩展到“数据概览、城市管理、景点管理”能力；概览页通过 `/api/admin/overview` 获取后端聚合数据，并使用 ECharts 展示用户、城市和景点统计。
- 管理端用户、城市、景点列表分页与主要筛选条件已对接后端查询参数，组件内不再做假分页切片。
- 管理端文本搜索统一由搜索按钮或回车提交，输入过程只更新草稿，不实时请求列表、城市候选或景点候选。
- 管理端新增城市使用单一城市/省份远程搜索框，候选确认后自动回填六位 `adcode` 和城市中心点；新增景点支持百度 POI 联想与地图选点，不再手填经纬度。

## 查阅指引

- 做页面布局前：查看原型图和 `docs/frontend/前端工程规范.md`。
- 做接口调用前：查看后端接口文档；如果不存在，应先补接口契约文档。
- 做地图相关功能前：查看 `docs/地图API接入说明.md`。
- 修改技术栈或目录结构前：先确认 `docs/开发规划.md` 是否需要同步更新。
