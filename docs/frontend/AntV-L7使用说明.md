# AntV L7 使用说明

## 使用边界

本项目已有百度地图工作台，用于路线规划、POI 选点、路径绘制和真实地图交互。AntV L7 只用于“地理数据可视化”，例如：

- 我的足迹打卡点分布。
- 城市维度聚合点。
- 后续热力图、飞线图、访问轨迹图。

不要用 L7 替代现有百度地图路线规划组件。涉及路线规划、地图选点、POI 搜索时，继续复用百度地图相关组件。

## 当前接入场景

当前用于 `/checkins` 我的足迹页面：

- 左侧 L7 地图展示当前筛选页的打卡点。
- 右侧保留时间轴和卡片列表，方便用户查看具体记录。
- 点击地图点位后，右侧记录进入选中态。

## 依赖

项目已安装：

```json
"@antv/l7": "^2.22.0"
```

当前优先使用 `@antv/l7` 内置的 `Map` 简单底图，避免足迹页再绑定第三方地图 Key。

## 组件封装原则

- L7 初始化必须放在独立组件内，不要散落在页面多个位置。
- L7 场景通过 `useEffect` 创建和销毁，组件卸载时必须调用 `scene.destroy()`。
- 图层数据从后端接口响应映射，不在组件内写 mock 点位。
- 地图点位点击只负责选择记录，不直接修改业务数据。
- 业务筛选、分页、排序仍由后端接口承担。

## 推荐数据结构

后端打卡接口返回的 `CheckinSpotItemDto` 已包含：

- `spotId`
- `name`
- `cityName`
- `type`
- `position.lng`
- `position.lat`
- `checkedInAt`
- `remark`
- `coverUrl`
- `tags`

L7 点图层使用前需要平铺坐标：

```ts
const layerData = spots.map((spot) => ({
  ...spot,
  lng: spot.position.lng,
  lat: spot.position.lat,
}));
```

## 视觉规范

- 足迹点默认使用蓝绿主色，历史/博物馆类可偏蓝，美食/夜游可偏橙。
- 选中点使用更大的点半径和明显描边。
- 地图不展示统计卡片；汇总数据后续放到独立个人主页。
- 地图页必须保留列表或卡片视图，不能只依赖地图查看数据。

## 验证要求

涉及 L7 的前端改动后至少执行：

```bash
npm run lint
npm run build
```

如果用户本地已启动开发服务，AI 不主动启动 `npm run dev`。
