# 行迹旅图 - 前端项目简历写法

## 项目定位

**行迹旅图** 是一个面向国内自由行用户的旅游地图规划 Web 应用。项目以地图工作台为核心，支持城市景点浏览、标签筛选、景点详情联动、行程池管理、真实路线规划、完整行程时间轴、收藏打卡和公开行程分享。

适合投递岗位：前端开发、Web 前端工程师、React 前端开发。

## 简历项目名称

推荐写法：

```text
行迹旅图 - 全国旅游地图规划平台
```

备选写法：

```text
全国旅游地图与行程规划系统
```

不建议写成“旅游网站”或“地图 Demo”，会弱化项目复杂度。

## 技术栈写法

```text
React 19、TypeScript、Vite、React Router、Ant Design、TanStack Query、CSS Modules、百度地图 JavaScript API、ECharts
```

如果简历空间紧张：

```text
React + TypeScript + Vite + TanStack Query + Ant Design + 百度地图 API
```

## 精简版

适合简历空间有限时使用：

```text
行迹旅图 - 全国旅游地图规划平台
技术栈：React、TypeScript、Vite、Ant Design、TanStack Query、百度地图 JS API

- 负责地图工作台核心功能开发，支持城市切换、景点搜索筛选、地图 Marker 展示、景点详情联动和行程池管理。
- 封装百度地图 SDK 加载、坐标转换、地图选点、路线 Polyline 绘制和路线端点标识，完成路线结果在地图上的可视化展示。
- 使用 TanStack Query 管理城市、景点、标签、路线规划、收藏/打卡状态等服务端状态，结合 enabled、placeholderData 优化请求触发和页面切换体验。
- 拆分 pages、components、hooks、api、lib、types、utils 等前端层次，降低地图工作台复杂页面的维护成本。
```

## 优化版

适合项目经历区重点展示：

```text
行迹旅图 - 全国旅游地图规划平台
项目描述：面向国内自由行场景的地图规划应用，支持用户按城市查看热门景点，在地图上筛选和查看详情，并将景点加入行程池生成自由路线或多天完整行程。
技术栈：React 19、TypeScript、Vite、React Router、Ant Design、TanStack Query、CSS Modules、百度地图 JavaScript API

- 主导地图工作台前端实现，完成城市切换、景点列表、标签筛选、景点详情、行程池、路线抽屉和完整行程时间轴等核心交互。
- 封装百度地图接入逻辑，包括 SDK 单例加载、AK 环境变量注入、GCJ-02/BD-09 坐标转换、Marker 渲染、地图选点、路线折线和区域轮廓展示。
- 基于 TanStack Query 统一管理服务端状态，按 cityId、spotId、筛选条件和登录态控制请求，避免未登录场景产生无效收藏/打卡请求。
- 设计路线结果前端展示模型，将后端返回的路线分段、总耗时、总距离、景点停留计划和 Day 时间轴映射为地图路线、右侧详情和底部行程状态。
- 使用 CSS Modules 和 Ant Design 组合实现响应式地图工作台，处理桌面端多栏布局和移动端内容换行、滚动、遮挡问题。
```

## 面试官友好版

适合“1-3 年前端”简历，表达更像真实工作项目：

```text
行迹旅图 - 全国旅游地图规划平台
项目描述：一个前后端分离的旅游地图规划系统，前端提供地图工作台、个人中心和管理后台；用户可以浏览城市景点、收藏/打卡景点、生成路线规划并保存或分享行程。
技术栈：React、TypeScript、Vite、React Router、Ant Design、TanStack Query、CSS Modules、百度地图 JS API、ECharts

- 负责地图工作台核心链路开发，将城市、景点、标签、路线规划、行程保存等接口接入前端，完成从“选城市-选景点-生成路线-保存分享”的闭环体验。
- 抽象百度地图适配层，处理 SDK 动态加载、地图初始化、点位创建、坐标系转换、路线绘制、起点选点和当前位置展示，避免地图能力散落在业务组件中。
- 使用 TanStack Query 封装查询和变更逻辑，支持条件请求、缓存复用、列表切换保留旧数据、登录态查询隔离，提升复杂页面的数据流稳定性。
- 将地图工作台拆分为 Header、BaiduMapStage、SpotRecommendList、SpotDetailPanel、TripPlannerDock、RoutePlanDrawer 等业务组件，页面层只负责状态编排。
- 完成收藏、打卡、我的行程、公开分享等用户态功能接入，请求层统一注入 Bearer Token，并处理加载、空数据、错误和未登录引导状态。
```

## 可强调的前端亮点

- **复杂交互不是简单表单**：地图、列表、详情、行程池、路线抽屉和时间轴之间有多处联动。
- **第三方地图 SDK 接入完整**：不只是显示地图，还包括坐标转换、Marker、Polyline、选点、定位、路线方向和端点标识。
- **服务端状态管理清晰**：TanStack Query 管理接口数据，React state 管理当前页面临时交互状态。
- **有真实联调场景**：接口统一响应、登录 Token、401 场景、保存行程、公开分享都能讲。
- **组件拆分有依据**：按业务区域拆组件，按数据层拆 api/hooks/lib/types。

## 不建议写的内容

- 不写“日活、用户量、转化率”等项目里没有的数据。
- 不写“高并发”“微服务”“分布式”。
- 不写“AI 自动生成旅行攻略”，当前项目主线是地图和路线规划，不是 AI 攻略。
- 不把百度地图路线规划写成自研算法；可以说“接入百度地图路线能力，并在前端完成路线可视化”。

## 30 秒项目介绍

```text
这个项目是一个旅游地图规划平台，我主要做前端部分。核心页面是地图工作台，用户可以切换城市、筛选景点、查看地图点位和景点详情，把想去的景点加入行程池后生成路线规划。前端接入了百度地图 JS API，做了 SDK 加载、坐标转换、Marker、Polyline 和地图选点；接口数据用 TanStack Query 管理，包括城市景点、路线规划、收藏打卡、我的行程和公开分享。这个项目的难点主要在地图、多组件联动和服务端状态管理。
```
