# 行迹旅图 - 前端项目简历写法

## 项目定位

**行迹旅图** 是一个面向国内自由行用户的旅游地图规划 Web 应用。项目以地图工作台为核心，支持城市景点浏览、标签筛选、景点详情联动、行程池管理、真实路线规划、完整行程时间轴、收藏打卡和公开行程分享。

适合投递岗位：前端开发、Web 前端工程师、React 前端开发。

## 简历项目名称

推荐写法：

```text
行迹旅图 - 全国旅游地图规划平台
```

备选写法：

```text
全国旅游地图与行程规划系统
```

不建议写成“旅游网站”或“地图 Demo”，会弱化项目复杂度。

## 技术栈写法

```text
React 19、TypeScript、Vite、React Router、Ant Design、TanStack Query、CSS Modules、百度地图 JavaScript API、ECharts
```

如果简历空间紧张：

```text
React + TypeScript + Vite + TanStack Query + Ant Design + 百度地图 API
```

如果偏前端可视化方向：

```text
React + TypeScript + 百度地图 JavaScript API + AntV L7 + ECharts + TanStack Query
```

## 精简版

适合简历空间有限时使用：

```text
行迹旅图 - 全国旅游地图规划平台
技术栈：React、TypeScript、Vite、Ant Design、TanStack Query、百度地图 JS API

- 解决地图工作台多区域联动问题，打通城市切换、景点筛选、地图点位、详情面板、行程池和路线结果之间的状态同步。
- 封装百度地图接入层，处理 SDK 单例加载、坐标转换、Marker / Polyline / 区域轮廓展示和地图选点，完成路线结果可视化。
- 使用 TanStack Query 管理城市、景点、标签、路线规划、收藏/打卡状态等服务端状态，结合 enabled、placeholderData 控制请求触发和切换体验。
- 支持完整行程模式展示，把自由路线和多天 Day 时间轴统一映射为前端可展示、可保存、可回放的数据结构。
```

## 优化版

适合项目经历区重点展示：

```text
行迹旅图 - 全国旅游地图规划平台
项目描述：面向国内自由行场景的地图规划应用，支持用户按城市查看热门景点，在地图上筛选和查看详情，并将景点加入行程池生成自由路线或多天完整行程。
技术栈：React 19、TypeScript、Vite、React Router、Ant Design、TanStack Query、CSS Modules、百度地图 JavaScript API

- 解决地图工作台的复杂联动问题：选中景点会同步影响地图点位、详情面板、收藏/打卡状态、行程池和路线结果展示。
- 封装百度地图接入逻辑，包括 SDK 单例加载、AK 环境变量注入、GCJ-02/BD-09 坐标转换、Marker 渲染、地图选点、路线折线和区域轮廓展示。
- 基于 TanStack Query 统一管理服务端状态，按 cityId、spotId、筛选条件和登录态控制请求，避免未登录场景产生无效收藏/打卡请求。
- 设计路线结果前端展示模型，将后端返回的路线分段、总耗时、总距离、景点停留计划和 Day 时间轴统一映射为地图路线、右侧详情和保存行程结构。
- 接入公开分享、行程回放和完整行程模式，使地图浏览、路线规划、行程保存与分享形成完整业务闭环。
```

## 面试官友好版

适合“1-3 年前端”简历，表达更像真实工作项目：

```text
行迹旅图 - 全国旅游地图规划平台
项目描述：一个前后端分离的旅游地图规划系统，前端提供地图工作台、个人中心和管理后台；用户可以浏览城市景点、收藏/打卡景点、生成路线规划并保存或分享行程。

- 抽象百度地图适配层，处理 SDK 动态加载、地图初始化、点位创建、坐标系转换、路线绘制、起点选点和当前位置展示，避免地图能力散落在业务组件中。

- 支持完整行程模式、多天 Day 时间轴和公开分享回放，统一自由路线与 itinerary 结果的前端展示和保存结构。
- 在足迹页引入 AntV L7 做省市聚合可视化，按场景区分主地图 SDK 和专题图层方案。
```

## 前端可视化方向写法

适合投递“前端可视化工程师”“数据可视化前端”“地图可视化方向前端”时使用：

```text
行迹旅图 - 全国旅游地图规划平台
项目描述：一个以地图交互和路线展示为核心的旅游规划系统，包含主地图工作台、路线结果可视化、完整行程时间轴，以及基于省市聚合数据的足迹专题可视化。
技术栈：React、TypeScript、百度地图 JavaScript API、AntV L7、ECharts、TanStack Query

- 负责地图可视化主链路，完成城市切换、景点点位、区域轮廓、路线折线、端点标识和地图选点等空间信息展示。
- 处理 GCJ-02 / BD-09 坐标转换、地图 SDK 单例加载和路线结果映射，保证景点、路线和交互位置的一致性。
- 将后端返回的路线分段、总耗时、总距离和多天 itinerary 结果映射为地图路线、时间轴和回放视图，统一可视化结果结构。
- 在足迹页引入 AntV L7 展示省级和城市级聚合分布，按专题图层场景补充地图数据表达能力。
- 管理后台使用 ECharts 展示用户、城市和景点统计，将业务数据聚合结果转成可阅读的图表摘要。
```

如果空间更紧，可以压缩成：

```text
- 负责地图与专题可视化实现，基于百度地图 JS API 完成景点点位、路线折线、区域轮廓和选点交互。
- 使用 AntV L7 实现足迹聚合地图，使用 ECharts 展示后台统计图表，补充专题数据表达。
- 统一路线分段、多天 itinerary 和分享回放的前端可视化结果结构，支撑地图、时间轴和回放联动。
```

## 可强调的前端亮点

- **复杂交互不是简单表单**：地图、列表、详情、行程池、路线抽屉和时间轴之间有多处联动。
- **第三方地图 SDK 接入完整**：不只是显示地图，还包括坐标转换、Marker、Polyline、选点、定位、路线方向和端点标识。
- **有地图与专题可视化分层思路**：主地图用百度地图承接底图和交互，专题足迹图层用 L7 补足聚合表达。
- **服务端状态管理清晰**：TanStack Query 管理接口数据，React state 管理当前页面临时交互状态。
- **有真实联调场景**：接口统一响应、登录 Token、401 场景、保存行程、公开分享都能讲。
- **结果模型有统一抽象**：自由路线、完整行程、保存行程和公开分享回放共用一套核心展示结构。

## 不建议写的内容

- 不写“日活、用户量、转化率”等项目里没有的数据。
- 不写“高并发”“微服务”“分布式”。
- 不写“AI 自动生成旅行攻略”，当前项目主线是地图和路线规划，不是 AI 攻略。
- 不把百度地图路线规划写成自研算法；可以说“接入百度地图路线能力，并在前端完成路线可视化”。

## 30 秒项目介绍

```text
这个项目是一个旅游地图规划平台，我主要做前端部分。核心页面是地图工作台，用户可以切换城市、筛选景点、查看地图点位和景点详情，把想去的景点加入行程池后生成路线规划。前端接入了百度地图 JS API，做了 SDK 加载、坐标转换、Marker、Polyline 和地图选点；接口数据用 TanStack Query 管理，包括城市景点、路线规划、收藏打卡、我的行程和公开分享。这个项目的难点主要在地图、多组件联动和服务端状态管理。
```
