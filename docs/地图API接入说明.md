# 地图 API 接入说明

第一阶段不接真实地图 API，前端先使用 Mock 地图容器和模拟点位。等页面结构、后端接口稳定后，再接入高德地图。

## 为什么先 Mock

- 地图 API 涉及 Key、域名、安全配置和额度限制。
- 先 Mock 可以专注完成页面结构、接口契约和数据模型。
- 后续替换为真实地图时，只需要替换地图组件内部实现。

## 后续需要你协助的事项

接入高德地图前，需要你完成以下准备：

1. 注册或登录高德开放平台。
2. 创建 Web 端应用。
3. 申请 Web JS API Key。
4. 配置安全域名：
   - 本地开发通常需要允许 `localhost` 或 `127.0.0.1`。
   - 正式部署后需要配置真实线上域名。
5. 如果使用服务端路线规划接口，还需要确认后端调用方式和配额。

## 前端环境变量规划

前端后续使用 `.env.local` 保存本地 Key：

```plain
VITE_AMAP_JS_KEY=你的高德Web端Key
VITE_AMAP_SECURITY_CODE=你的安全密钥
```

真实 Key 不应写死在源码里，也不应提交到仓库。

## 后端环境变量规划

如果后端调用高德 Web 服务 API，后续使用环境变量或配置文件占位：

```plain
AMAP_WEB_SERVICE_KEY=你的高德服务端Key
```

后端负责路线规划时，接口可以先设计为：

```plain
POST /api/routes/plan
```

请求内容包含城市、起点、景点列表、交通方式和规划模式。返回内容包含路线总览和每一段路线。

## MVP 接入顺序

1. 前端加载高德地图 JS API。
2. 根据城市中心点定位地图。
3. 根据景点经纬度绘制 Marker。
4. 点击 Marker 后联动右侧景点详情。
5. 后端提供路线规划接口。
6. 前端调用后端路线规划接口并渲染路线分段。

## 风险提醒

- 地图 Key 泄露会带来额度消耗风险。
- 前端 Key 需要配置安全域名，后端 Key 不要暴露给浏览器。
- 路线规划结果依赖第三方 API，失败时需要有降级提示。
- 多端扩展时，App 和小程序可能需要分别申请不同类型的 Key。
