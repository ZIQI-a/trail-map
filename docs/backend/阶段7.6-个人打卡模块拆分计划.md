# 阶段 7.6：个人打卡模块拆分计划

## 当前结论

个人打卡模块基于现有用户体系继续开发。当前项目已经完成登录、收藏和我的行程，且 `V8__create_user_domain_tables.sql` 已预留 `user_checkin_spot` 表，因此第一步不新增数据库迁移，直接补后端接口。

## 产品边界

- 打卡表示“我去过这个景点”，不是上班考勤。
- 第一版不做定位距离校验，不要求用户必须在景点附近。
- 可记录用户主动提交的打卡坐标和备注，但坐标只作为展示/回忆信息。
- 一个用户对同一个景点只保留一条打卡记录，重复打卡按更新备注和坐标处理。

## 7.6.1 后端打卡基础接口

### 目标

建立打卡后端最小闭环，让前端可以查询状态、打卡、取消打卡和展示我的足迹列表。

### 交付物

- 实体：`UserCheckinSpot`
- Mapper：`UserCheckinSpotMapper`
- 服务：`CheckinSpotService`
- 控制器：`CheckinSpotController`
- 接口：
  - `GET /api/checkin-spots/{spotId}/status`
  - `POST /api/checkin-spots/{spotId}`
  - `DELETE /api/checkin-spots/{spotId}`
  - `GET /api/checkin-spots`

### 验收标准

- 未登录访问打卡接口返回 401。
- 已登录用户可以打卡、查询状态、查看列表和取消打卡。
- 重复打卡不会产生重复记录。
- 打卡只影响当前用户，不修改公共景点数据。
- `mvn test` 通过。

## 7.6.2 前端景点详情接入

### 目标

在景点详情面板接入“去过/已打卡”按钮。

### 交付物

- 景点详情查询当前用户打卡状态。
- 未登录点击打卡时弹出登录框。
- 已登录可打卡和取消打卡。

### 验收标准

- 登录前后按钮状态清晰。
- 打卡成功后当前景点状态立即刷新。

## 7.6.3 我的足迹页面

### 目标

新增“我的足迹”页面，用于展示用户打卡过的景点。

### 交付物

- 路由：`/checkins`
- 足迹卡片列表。
- 城市、标签、时间范围筛选。
- 从用户头像菜单进入。

### 验收标准

- 可以看到已打卡景点的图片、城市、打卡时间和备注。
- 支持空状态和未登录提示。

## 7.6.4 地图和列表联动增强

### 目标

让打卡状态影响地图工作台体验。

### 可选能力

- 景点卡片展示“已去过”标识。
- 地图 marker 展示已打卡状态。
- 增加“隐藏已去景点”开关。
- 城市足迹统计：已打卡数量 / 当前城市景点总数。

## 教学说明

打卡和收藏都是“用户与景点的关系表”。区别是收藏偏向未来计划，打卡偏向历史足迹，所以打卡表多了 `checkin_time`、可选坐标和备注。先做后端接口，是为了让前端状态有明确的数据来源。
