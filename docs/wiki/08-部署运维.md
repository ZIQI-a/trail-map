## 环境配置

### 环境变量清单

#### 后端环境变量

| 变量名 | 说明 | 默认值 | 必填 |
| --- | --- | --- | --- |
| `TRAILMAP_DB_HOST` | MySQL 主机地址 | `localhost` | 生产环境必填 |
| `TRAILMAP_DB_PORT` | MySQL 端口 | `3306` | 否 |
| `TRAILMAP_DB_NAME` | 数据库名称 | `trailmap` | 否 |
| `TRAILMAP_DB_USERNAME` | 数据库用户名 | `root` | 生产环境必填 |
| `TRAILMAP_DB_PASSWORD` | 数据库密码 | - | 生产环境必填 |
| `BAIDU_MAP_SERVER_AK` | 百度地图服务端 AK | - | 路线规划功能必填 |
| `TRAILMAP_AUTH_TOKEN_SECRET` | Token 签名密钥 | `trailmap-dev-secret` | 生产环境必须修改 |

#### 前端环境变量

| 变量名 | 说明 | 默认值 |
| --- | --- | --- |
| `VITE_API_BASE_URL` | 后端 API 地址 | `""`（同源） |
| `VITE_BAIDU_MAP_AK` | 百度地图前端 AK | - |

### 环境变量配置方式

#### 后端

方式一：系统环境变量
```bash
export TRAILMAP_DB_HOST=production-db-host
export TRAILMAP_DB_PASSWORD=secure_password
```

方式二：`.env.local.properties` 文件（项目根目录）
```properties
TRAILMAP_DB_HOST=production-db-host
TRAILMAP_DB_PASSWORD=secure_password
```

#### 前端

创建 `.env.local` 文件（`frontend/` 目录）：
```env
VITE_API_BASE_URL=https://api.trailmap.example.com
VITE_BAIDU_MAP_AK=your_production_ak
```

## 后端部署

### 构建可执行 JAR

```bash
cd backend
mvn clean package -DskipTests
```

生成文件：`backend/target/trailmap-backend-0.0.1-SNAPSHOT.jar`

### 运行

```bash
java -jar target/trailmap-backend-0.0.1-SNAPSHOT.jar \
  --server.port=8080 \
  --TRAILMAP_DB_HOST=production-host \
  --TRAILMAP_DB_PASSWORD=secure_password
```

### Docker 部署（推荐）

创建 `backend/Dockerfile`：

```dockerfile
FROM eclipse-temurin:21-jre-alpine
WORKDIR /app
COPY target/*.jar app.jar
EXPOSE 8080
ENTRYPOINT ["java", "-jar", "app.jar"]
```

构建与运行：
```bash
# 构建镜像
docker build -t trailmap-backend ./backend

# 运行容器
docker run -d \
  --name trailmap-backend \
  -p 8080:8080 \
  -e TRAILMAP_DB_HOST=db-host \
  -e TRAILMAP_DB_PASSWORD=secure_password \
  -e BAIDU_MAP_SERVER_AK=your_ak \
  trailmap-backend
```

### Docker Compose（完整栈）

创建 `docker-compose.yml`：

```yaml
version: '3.8'
services:
  db:
    image: mysql:8.0
    environment:
      MYSQL_ROOT_PASSWORD: root_password
      MYSQL_DATABASE: trailmap
    ports:
      - "3306:3306"
    volumes:
      - mysql-data:/var/lib/mysql

  backend:
    build: ./backend
    ports:
      - "8080:8080"
    environment:
      TRAILMAP_DB_HOST: db
      TRAILMAP_DB_USERNAME: root
      TRAILMAP_DB_PASSWORD: root_password
      BAIDU_MAP_SERVER_AK: ${BAIDU_MAP_SERVER_AK}
      TRAILMAP_AUTH_TOKEN_SECRET: ${AUTH_SECRET}
    depends_on:
      - db

  frontend:
    build: ./frontend
    ports:
      - "80:80"
    depends_on:
      - backend

volumes:
  mysql-data:
```

## 前端部署

### 构建生产版本

```bash
cd frontend
npm run build
```

生成目录：`frontend/dist/`

### Nginx 配置示例

```nginx
server {
    listen 80;
    server_name trailmap.example.com;

    # 前端静态资源
    location / {
        root /usr/share/nginx/html;
        index index.html;
        try_files $uri $uri/ /index.html;
    }

    # API 代理
    location /api/ {
        proxy_pass http://backend:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

### 前端 Dockerfile

```dockerfile
FROM node:18-alpine AS build
WORKDIR /app
COPY package*.json ./
RUN npm ci
COPY . .
RUN npm run build

FROM nginx:alpine
COPY --from=build /app/dist /usr/share/nginx/html
COPY nginx.conf /etc/nginx/conf.d/default.conf
EXPOSE 80
CMD ["nginx", "-g", "daemon off;"]
```

## 数据库运维

### Flyway 迁移管理

Flyway 在应用启动时自动执行未运行的迁移脚本。

#### 手动迁移（如需要）

```bash
# 使用 Maven 插件
mvn flyway:migrate

# 查看迁移状态
mvn flyway:info
```

#### 迁移脚本规范

- 文件命名：`V{version}__{description}.sql`
- 版本号递增，不可修改已执行的脚本
- 编码：UTF-8
- 位置：`backend/src/main/resources/db/migration/`

### 数据库维护

```sql
-- 查看表状态
SHOW TABLE STATUS FROM trailmap;

-- 查看索引
SHOW INDEX FROM spot;

-- 分析慢查询
SHOW PROCESSLIST;
```

## 监控与日志

### 健康检查

后端提供健康检查接口：
```
GET /api/health
```

### 应用日志

Spring Boot 默认日志输出到 stdout，可通过以下方式配置：

```yaml
# application.yml
logging:
  level:
    com.trailmap: INFO
    org.springframework.security: WARN
```

### 生产环境日志建议

| 日志级别 | 使用场景 |
| --- | --- |
| ERROR | 异常、服务不可用 |
| WARN | 非关键问题、降级处理 |
| INFO | 关键业务流程、请求处理 |
| DEBUG | 开发调试（生产环境关闭） |

## 安全加固

### 必须项

1. **修改默认密码**
   - 数据库密码
   - Token 签名密钥 (`TRAILMAP_AUTH_TOKEN_SECRET`)

2. **HTTPS 配置**
   - 生产环境必须使用 HTTPS
   - Nginx 配置 SSL 证书

3. **数据库安全**
   - 使用专用数据库用户，不使用 root
   - 限制数据库访问 IP

4. **环境变量管理**
   - 敏感信息不写入代码或配置文件
   - 使用环境变量或密钥管理服务

### 推荐项

| 措施 | 说明 |
| --- | --- |
| CORS 配置 | 限制允许的来源域名 |
| 请求频率限制 | 防止恶意请求 |
| SQL 注入防护 | MyBatis-Plus 参数化查询已内置 |
| XSS 防护 | 前端输出转义 |
| CSRF 防护 | 无状态 Token 认证已规避 |

## 备份与恢复

### 数据库备份

```bash
# 全量备份
mysqldump -u root -p trailmap > backup_$(date +%Y%m%d).sql

# 压缩备份
mysqldump -u root -p trailmap | gzip > backup_$(date +%Y%m%d).sql.gz
```

### 数据库恢复

```bash
# 从备份恢复
mysql -u root -p trailmap < backup_20260101.sql

# 从压缩备份恢复
gunzip < backup_20260101.sql.gz | mysql -u root -p trailmap
```

### 定期备份策略

| 备份类型 | 频率 | 保留时间 |
| --- | --- | --- |
| 全量备份 | 每天 | 7 天 |
| 增量备份 | 每小时 | 24 小时 |
| 异地备份 | 每周 | 30 天 |
