## 环搭搭建

### 必要工具

| 工具 | 版本要求 | 用途 |
| --- | --- | --- |
| JDK | 21+ | 后端运行 |
| Maven | 3.8+ | 后端构建 |
| Node.js | 18+ | 前端运行 |
| npm | 9+ | 前端包管理 |
| MySQL | 8.0+ | 数据库 |
| Git | 2.x | 版本控制 |

### 后端环境

1. **安装 JDK 21**
   ```bash
   # 验证安装
   java -version
   ```

2. **配置数据库**
   - 创建 MySQL 数据库 `trailmap`
   - 配置环境变量或 `.env.local` 文件：
     ```properties
     TRAILMAP_DB_HOST=localhost
     TRAILMAP_DB_PORT=3306
     TRAILMAP_DB_NAME=trailmap
     TRAILMAP_DB_USERNAME=root
     TRAILMAP_DB_PASSWORD=your_password
     ```

3. **配置百度地图 AK**（可选，路线规划功能需要）
   ```properties
   BAIDU_MAP_SERVER_AK=your_baidu_map_ak
   ```

4. **安装 Maven 依赖**
   ```bash
   cd backend
   mvn clean install -DskipTests
   ```

### 前端环境

1. **安装 Node.js 依赖**
   ```bash
   cd frontend
   npm install
   ```

2. **配置环境变量**（可选）
   
   创建 `frontend/.env.local`：
   ```env
   VITE_API_BASE_URL=http://localhost:8080
   VITE_BAIDU_MAP_AK=your_baidu_map_ak
   ```

## 项目启动

### 后端启动

```bash
cd backend
mvn spring-boot:run
```

后端默认运行在 `http://localhost:8080`。

启动后 Flyway 会自动执行数据库迁移，无需手动建表。

### 前端启动

```bash
cd frontend
npm run dev
```

前端默认运行在 `http://localhost:5173`。

### API 文档

后端启动后访问 Swagger UI：
```
http://localhost:8080/swagger-ui.html
```

## 代码规范

### 后端规范

| 规范项 | 要求 |
| --- | --- |
| 语言 | Java 21 |
| 包结构 | `com.trailmap.{common\|config\|controller\|entity\|mapper\|model\|service}` |
| 分层职责 | Controller 不写业务逻辑；Service 不直接操作数据库；Mapper 只负责 SQL |
| 注释 | 类、方法和重要业务步骤必须添加中文注释 |
| 命名 | 类名 PascalCase，方法/变量 camelCase，常量 UPPER_SNAKE_CASE |
| API 前缀 | 所有接口以 `/api` 开头 |
| 响应格式 | 统一使用 `ApiResponse<T>` 包装 |
| 数据库迁移 | 已执行的 V* 脚本禁止修改，变更必须新增版本 |

### 前端规范

| 规范项 | 要求 |
| --- | --- |
| 语言 | TypeScript 5.8 |
| 目录结构 | `pages/` (页面) / `components/` (组件) / `api/` (接口) / `hooks/` (查询) / `lib/` (基础) |
| 组件规范 | Props 必须声明 TypeScript 类型；状态放在最小需要范围内 |
| 样式方案 | 业务组件优先使用 CSS Modules；通用 UI 使用 Ant Design |
| 接口规范 | 所有请求统一放在 `api/`，通用封装在 `lib/http.ts` |
| 地图规范 | 百度地图 AK 通过 `.env.local` 注入，禁止硬编码 |
| 注释 | 组件函数需要添加简短中文注释说明职责 |

### 命名约定

**后端**：
```
Controller:  {Domain}Controller       (CityController, SpotController)
Service:     {Domain}Service          (CityService, SpotService)
ServiceImpl: {Domain}ServiceImpl      (CityServiceImpl, SpotServiceImpl)
Mapper:      {Domain}Mapper           (CityMapper, SpotMapper)
Entity:      {EntityName}             (City, Spot, AppUser)
```

**前端**：
```
页面组件:    {Name}Page.tsx           (MapWorkbenchPage, AdminPage)
业务组件:    {Name}.tsx               (BaiduMapStage, SpotDetailPanel)
样式文件:    {Name}.module.css        (对应组件的 CSS Module)
Hook:        use{Name}.ts             (useMapWorkbenchData, useAdminData)
API 模块:    {domain}.ts              (mapWorkbench.ts, auth.ts)
```

## 开发工作流

### 功能开发流程

1. **确认需求**：查看 `docs/开发规划.md` 确认当前阶段目标
2. **查看设计文档**：阅读相关需求文档和数据库设计
3. **编写代码**：遵循前后端工程规范
4. **执行质量门禁**：运行 lint 和 build
5. **同步文档**：更新相关接口、数据库或配置说明文档

### 文档同步要求

当代码改动涉及以下内容时，必须同步更新对应文档：

| 变更类型 | 需同步的文档 |
| --- | --- |
| 接口变更 | `docs/backend/后端工程规范.md`, API 文档 |
| 数据库变更 | `require/数据库设计.md`, `docs/backend/数据库迁移版本说明.md` |
| 配置变更 | 相关 `.env` 示例文件 |
| 前端目录变更 | `docs/frontend/前端工程规范.md` |

## 测试策略

### 后端测试

```bash
# 运行全部测试
mvn test

# 跳过测试编译
mvn package -DskipTests
```

### 前端测试

```bash
# 代码检查
npm run lint

# 类型检查 + 生产构建
npm run build
```

### 联调验证

当前后端改动涉及接口变更时，需要同时验证：
1. 后端接口返回正确
2. 前端能正常调用新接口
3. `npm run build` 无报错

## Git 工作流

### 分支管理

| 分支 | 用途 |
| --- | --- |
| `main` | 生产分支，保持可部署状态 |
| `dev` | 开发分支，功能合并到此 |
| `feature/*` | 功能分支，从 dev 分出 |
| `fix/*` | 修复分支 |

### 提交规范

```
<type>(<scope>): <subject>

# 示例
feat(backend): 新增用户收藏接口
fix(frontend): 修复地图标注闪烁问题
docs(wiki): 更新 API 文档
refactor(service): 重构路线规划服务
```

**Type 类型**：
- `feat`: 新功能
- `fix`: Bug 修复
- `docs`: 文档变更
- `refactor`: 代码重构
- `test`: 测试相关
- `chore`: 构建/工具变更

## 调试技巧

### 后端调试

1. **查看 SQL 日志**

   在 `application.yml` 中配置：
   ```yaml
   mybatis-plus:
     configuration:
       log-impl: org.apache.ibatis.logging.stdout.StdOutImpl
   ```

2. **Swagger UI 测试**
   
   访问 `http://localhost:8080/swagger-ui.html` 直接测试接口

3. **查看 Flyway 迁移状态**
   
   检查 `flyway_schema_history` 表

### 前端调试

1. **React DevTools**
   
   安装浏览器扩展 React DevTools

2. **网络请求**
   
   浏览器 DevTools Network 面板查看 API 调用

3. **状态查看**
   
   使用 TanStack Query DevTools 查看缓存状态

## 质量门禁

每次功能改动后必须通过以下检查：

### 后端

```bash
# 运行测试
mvn test

# 检查编译
mvn compile
```

### 前端

```bash
# 代码检查
npm run lint

# 类型检查 + 构建
npm run build
```

### 检查清单

- [ ] 后端 `mvn test` 通过
- [ ] 前端 `npm run lint` 无错误
- [ ] 前端 `npm run build` 成功
- [ ] 接口文档已更新（如有接口变更）
- [ ] 数据库文档已更新（如有表结构变更）
- [ ] 中文注释已添加到新增代码
