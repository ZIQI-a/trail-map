## 启动问题

### 后端启动失败

#### 端口被占用

**错误信息**：
```
Web server failed to start. Port 8080 was already in use.
```

**解决方案**：
```bash
# 查找占用端口的进程
netstat -ano | findstr :8080

# 或使用 PowerShell
Get-NetTCPConnection -LocalPort 8080

# 终止进程
taskkill /PID <进程ID> /F
```

#### 数据库连接失败

**错误信息**：
```
Communications link failure
```

**排查步骤**：
1. 确认 MySQL 服务已启动
2. 检查数据库连接参数（host、port、username、password）
3. 确认数据库 `trailmap` 已创建
4. 检查网络连通性

```bash
# 测试数据库连接
mysql -h localhost -P 3306 -u root -p trailmap
```

#### Flyway 迁移失败

**错误信息**：
```
Validate failed: Migrations have failed validation
```

**排查步骤**：
1. 检查 `flyway_schema_history` 表
2. 确认迁移脚本文件名正确（`V{version}__{desc}.sql`）
3. 检查 SQL 语法错误

```sql
-- 查看迁移历史
SELECT * FROM flyway_schema_history ORDER BY installed_rank DESC;
```

#### JDK 版本不兼容

**错误信息**：
```
Unsupported class file major version
```

**解决方案**：
```bash
# 检查 Java 版本
java -version

# 确保使用 JDK 21+
```

## 数据库问题

### 表不存在

**错误信息**：
```
Table 'trailmap.xxx' doesn't exist
```

**解决方案**：
1. 确认 Flyway 迁移已执行
2. 检查数据库名称是否正确
3. 手动执行缺失的迁移脚本

### 外键约束错误

**错误信息**：
```
Cannot add or update a child row: a foreign key constraint fails
```

**排查步骤**：
1. 确认引用的父表记录存在
2. 检查外键字段值是否正确
3. 确认数据插入顺序（先父表后子表）

### 唯一约束冲突

**错误信息**：
```
Duplicate entry 'xxx' for key 'uk_xxx'
```

**解决方案**：
1. 检查是否重复插入
2. 使用 `INSERT ... ON DUPLICATE KEY UPDATE` 或 `REPLACE INTO`
3. 先查询再决定插入或更新

## 接口问题

### 404 Not Found

**排查步骤**：
1. 确认请求路径正确（以 `/api` 开头）
2. 检查 Controller 的 `@RequestMapping` 注解
3. 确认 HTTP 方法匹配（GET/POST/PUT/DELETE）

### 400 Bad Request

**排查步骤**：
1. 检查请求体格式（JSON）
2. 查看参数校验错误信息
3. 确认 Content-Type 设置为 `application/json`

### 500 Internal Server Error

**排查步骤**：
1. 查看后端日志输出
2. 检查数据库连接状态
3. 查看是否为空指针异常
4. 检查服务依赖是否正常

### 响应数据异常

**排查步骤**：
1. 使用 Swagger UI 测试接口
2. 检查 Entity 字段映射是否正确
3. 确认 SQL 查询结果

## 前端问题

### npm install 失败

**解决方案**：
```bash
# 清除缓存
npm cache clean --force

# 删除 node_modules 重新安装
rm -rf node_modules package-lock.json
npm install

# 使用国内镜像
npm install --registry=https://registry.npmmirror.com
```

### TypeScript 类型错误

**排查步骤**：
1. 检查类型定义文件 `types/`
2. 确认 API 响应类型与后端一致
3. 运行 `npm run build` 查看详细错误

### 组件渲染异常

**排查步骤**：
1. 打开浏览器 DevTools Console
2. 检查 React 错误边界
3. 确认 props 类型正确
4. 检查条件渲染逻辑

### 页面空白

**排查步骤**：
1. 检查路由配置是否正确
2. 确认组件导入路径
3. 查看控制台错误信息
4. 检查网络请求是否成功

## 地图相关问题

### 地图不显示

**排查步骤**：
1. 检查百度地图 AK 是否配置
2. 确认 AK 是否有效且未过期
3. 检查控制台是否有加载错误
4. 确认网络可以访问百度地图服务

```javascript
// 检查 AK 配置
console.log(import.meta.env.VITE_BAIDU_MAP_AK);
```

### 坐标偏移

**说明**：
- 百度地图使用 BD09 坐标系
- GPS 使用 WGS84 坐标系
- 国内地图使用 GCJ-02 坐标系

**解决方案**：
项目内置坐标转换工具 `utils/map-workbench/coordinate.ts`，确保使用正确的转换函数。

### 路线规划失败

**排查步骤**：
1. 检查 `BAIDU_MAP_SERVER_AK` 环境变量
2. 确认起点和终点坐标有效
3. 查看后端日志中的百度 API 响应
4. 检查百度地图服务配额

## 认证与权限问题

### 401 Unauthorized

**排查步骤**：
1. 检查 Token 是否存在（localStorage 中的 `trailmap_auth_token`）
2. 确认 Token 格式正确（Bearer 前缀）
3. 检查 Token 是否过期
4. 重新登录获取新 Token

### 403 Forbidden

**说明**：当前用户角色无权访问该接口

**排查步骤**：
1. 确认用户类型（normal / admin）
2. 检查接口权限配置
3. 管理员接口需要 ADMIN 角色

### 登录失败

**排查步骤**：
1. 确认用户名和密码正确
2. 检查用户状态是否启用
3. 查看后端日志中的认证错误

## 性能问题

### 接口响应慢

**排查步骤**：
1. 使用 Swagger UI 测试接口响应时间
2. 检查数据库查询是否优化
3. 确认索引是否创建
4. 查看是否有 N+1 查询问题

### 前端加载慢

**排查步骤**：
1. 使用浏览器 Network 面板分析加载时间
2. 检查是否有大量 API 请求
3. 确认懒加载配置
4. 检查图片资源大小

### 地图渲染卡顿

**排查步骤**：
1. 减少同时显示的标注数量
2. 使用聚合显示
3. 检查是否频繁重绘
4. 优化坐标数据量

## 常见问题速查表

| 问题 | 可能原因 | 解决方案 |
| --- | --- | --- |
| 后端启动报端口占用 | 8080 端口被占用 | 终止占用进程或修改端口 |
| 数据库连接失败 | MySQL 未启动或配置错误 | 检查 MySQL 服务和连接参数 |
| Flyway 迁移失败 | SQL 脚本错误或版本冲突 | 检查脚本语法和迁移历史 |
| 前端请求 404 | API 路径错误 | 确认路径以 `/api` 开头 |
| 前端请求 401 | Token 缺失或无效 | 重新登录获取 Token |
| 地图不显示 | AK 未配置或无效 | 配置有效的百度地图 AK |
| 类型错误 | 前后端类型不一致 | 同步 TypeScript 类型定义 |
| 构建失败 | 依赖问题或代码错误 | 清除缓存重新安装依赖 |
